# AGENTS.md — 学生管理系统开发约束

## 项目目标
基于单体架构的学生管理系统，用于课程实验，最终上传到 Gitee 和 GitHub。

## 架构
- 单体架构，单模块 Maven 项目。
- 不使用微服务、Nacos、Gateway、Redis，除非用户明确要求。

## 技术栈
- Java 8
- Spring Boot
- Maven
- MySQL
- Thymeleaf

## 开发规范
1. **先理解，再给方案** — 每次修改前先理解需求，给出最小实现方案，确认后再动手。
2. **单步开发** — 每次只完成一个明确功能，不要一次性大改。
3. **最小方案优先** — 避免引入不必要的抽象或依赖。
4. **不过度设计** — 只实现当前需求，不为"可能的未来需求"预留扩展点。
5. **不改无关文件** — 修改范围严格限定在需求涉及的类/文件，不顺手重构、不调整格式。
6. **不要随意引入新依赖**。
7. **不要修改 Codex 安装目录** — 所有项目代码、配置文件、README 必须写在当前 Git 仓库中。
8. **不要提交敏感信息** — target/、.idea/、*.iml、数据库密码、本地缓存文件不得提交到仓库。
9. **完成说明** — 每次修改后，说明改了哪些文件、为什么改、如何运行验证。

## 验证检查
每次完成阶段性功能后，优先检查：
- Maven 是否能正常加载依赖
- 项目是否能启动
- 页面是否能正常访问
- CRUD 功能是否能操作数据库
- README 是否说明运行步骤

## 项目定位补充
当前阶段不是完成最终课程设计成品，而是先开发一个通用后台管理骨架。不要加入具体学校名称、学校 logo、学校背景图、学校介绍等学校定制内容。

## 系统角色设计
系统只有两个角色：
1. **学生用户** — 可以注册，注册后默认是学生角色。
2. **管理员用户** — 同时代表教师用户，账号由初始化 SQL 提前插入数据库，不开放管理员注册。
登录后根据角色进入不同的页面和菜单。

## 系统功能模块
系统骨架包含以下模块：
1. 注册登录
2. 学生信息管理
3. 班级信息管理
4. 课程信息管理
5. 选课信息管理
6. 考勤信息管理
7. 请假系统管理
8. 成绩信息管理
9. 系统管理

## 角色权限设计

### 学生用户权限
- **学生信息**：只能查看自己的信息，只能修改头像，不能修改学号/姓名/班级/手机号/邮箱等核心信息，不能删除。
- **班级信息**：只能浏览，不能新增/修改/删除。
- **课程信息**：只能浏览，不能新增/修改/删除。
- **选课信息**：可以选课、查看自己的选课结果，不能管理其他学生的选课记录。
- **考勤信息**：只能查看自己的考勤记录，不能新增/修改/删除。
- **请假系统**：可以提交请假申请、查看自己的请假记录，不能审批，不能修改或删除他人记录。
- **成绩信息**：只能查看自己的成绩，不能新增/修改/删除。
- **系统管理**：可以修改密码、退出登录、管理自己的头像。

### 管理员用户权限
管理员拥有以下模块的增删改查权限：学生信息、班级信息、课程信息、选课信息、考勤信息、请假信息（含审批）、成绩信息、系统基础信息。管理员账号不通过注册产生，必须由初始化 SQL 插入。

* 输入成绩
* 填写备注
* 提交时校验成绩必须在 0 到 100 之间
* 同一个学生同一门课程不允许重复录入成绩

编辑成绩交互要求：

* 只允许修改成绩和备注
* 不建议修改学生和课程

删除成绩交互要求：

* 删除前需要确认
* 确认提示内容类似：确认删除该学生的这门课程成绩吗？

查看统计交互要求：

* 成绩列表页面可以提供"查看统计"入口
* 点击后进入成绩统计页面
* 如果从某门课程进入统计页面，可以自动带上课程筛选条件

### 2. 成绩统计页面

成绩统计页面第一版只做基础统计，不做复杂图表。

要求：

* 支持按课程筛选
* 支持按班级筛选
* 支持按考试类型筛选
* 显示平均分
* 显示最高分
* 显示最低分
* 显示及格率
* 显示成绩区间柱状图

成绩区间柱状图区间：

* 60 分以下
* 60-70 分
* 70-80 分
* 80-90 分
* 90-100 分

第一版暂时不做：

* 饼图
* 排行榜
* 复杂统计报表
* 复杂可视化大屏

## 补充六：成绩表设计补充

当前 `grade_info` 表需要能够支撑成绩列表和成绩统计页面。

建议字段：

| 字段名          | 说明                 |
| ------------ | ------------------ |
| id           | 主键                 |
| student_id   | 学生 ID              |
| student_no   | 学号，冗余保存，方便查询和展示    |
| student_name | 学生姓名，冗余保存，方便展示     |
| class_id     | 班级 ID              |
| class_name   | 班级名称，冗余保存，方便展示     |
| course_id    | 课程 ID              |
| course_name  | 课程名称，冗余保存，方便展示     |
| exam_type    | 考试类型：期中考试、期末考试、平时成绩 |
| score        | 成绩                 |
| grade_level  | 等级：优秀、良好、中等、及格、不及格 |
| status       | 状态：已录入、需关注         |
| remark       | 备注                 |
| create_time  | 录入时间               |
| update_time  | 更新时间               |

成绩等级规则：

* 90-100：优秀
* 80-89：良好
* 70-79：中等
* 60-69：及格
* 0-59：不及格

成绩状态规则：

* 60 分以下：需关注
* 60 分及以上：已录入

约束规则：

* 分数必须在 0 到 100 之间
* 同一个学生同一门课程同一考试类型不能重复录入成绩 → `UNIQUE(student_id, course_id, exam_type)`
* 编辑成绩时只允许修改成绩和备注
